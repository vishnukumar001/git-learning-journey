Git Basics Guide

Git is a distributed version control system used to track changes in source code during software development. It helps developers collaborate, manage versions, and restore previous versions of their projects.

Why Learn Git?

  • Track every change in your project.
  • Collaborate with other developers.
  • Restore previous versions of files.
  • Manage multiple features using branches.
  • Publish projects on GitHub.

Common Git Commands

Command Description
git init Initialize a new Git repository.
git clone Copy an existing repository.
git status Check the current repository status.
git add . Add all changed files.
git commit -m "message" Save changes with a commit message.
git push Upload commits to GitHub.
git pull Download the latest changes.
git branch List all branches.
git checkout branch-name Switch to another branch.
git merge Merge one branch into another.

Git Workflow

  1. Create or modify files.
  2. Check changes using git status.
  3. Add files using git add ..
  4. Commit changes using git commit.
  5. Push the project to GitHub using git push.
